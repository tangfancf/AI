# 生成测试用例 Skill

## 用途
根据需求文档（md格式），可选配合思维导图图片，自动生成标准化的测试用例 Excel 文件。

## 输入
用户需要提供：
1. **需求文档路径**（必填）：md 文档在电脑中的绝对路径
2. **思维导图图片路径**（选填）：思维导图图片在电脑中的绝对路径（支持 png/jpg 等格式）

## 输出
生成一个 `.xlsx` 格式的测试用例文件，保存到用户桌面。

## 用例字段顺序
| 列 | 字段名 | 说明 |
|----|--------|------|
| A | 编号 | TC-001 格式递增 |
| B | 用例标题 | 格式为【所属模块】+ 标题描述 |
| C | 前置条件 | 执行该用例前需要满足的条件 |
| D | 步骤 | 具体操作步骤 |
| E | 预期结果 | 期望的执行结果 |
| F | 用例等级 | P1（核心流程/主路径）/ P2（分支/边界/异常） |

## 用例标题规范
- 标题必须包含【所属模块】前缀
- 格式：`【模块名称】具体描述`
- 示例：`【P1-销售黑名单】手机号在黑名单中触发硬拦截`

## 用例等级划分
- **P1**：核心业务流程、主路径功能验证、关键拦截逻辑
- **P2**：分支场景、边界值、异常容错、UI细节、配置变更

## 执行流程

### Step 1: 读取输入
1. 读取用户提供的 md 需求文档内容
2. 如果用户提供了思维导图图片，读取并通过图片识别分析功能模块和流程

### Step 2: 分析需求
1. 从 md 文档中提取功能模块、业务规则、交互逻辑
2. 如果有思维导图，从中提取模块结构、流程分支、优先级关系，与文档信息综合分析
3. 如果没有思维导图，仅基于需求文档梳理完整的测试场景

### Step 3: 设计用例
按以下维度覆盖测试场景：
- **正向功能**：主流程、各模块核心功能
- **分支路径**：条件判断的各分支
- **优先级组合**：多条件同时命中时的优先级验证
- **边界值**：阈值临界点、上下限
- **异常容错**：接口超时、服务不可用、并发
- **UI交互**：弹窗格式、按钮状态、操作反馈
- **配置管理**：增删改查、权限控制
- **兼容性**：升级后老数据、多入口一致性

### Step 4: 生成 Excel
1. 使用 openpyxl 生成 xlsx 文件
2. 应用统一样式：表头蓝底白字、内容左对齐自动换行、细边框
3. 冻结首行
4. 文件命名格式：`SCRM-{功能名称}测试用例-{版本号}.xlsx`
5. 默认保存到用户桌面 `/Users/mac/Desktop/`

### Step 5: 输出结果
告知用户文件保存路径和用例总数。

## 示例交互

### 示例1：同时提供需求文档和思维导图

用户输入：
```
需求文档：<需求文档绝对路径>
思维导图：<思维导图绝对路径>
```

输出：
```
需求文档：<回显用户传入的需求文档路径>
思维导图：<回显用户传入的思维导图路径>
测试用例文件已保存到：<实际生成的xlsx文件路径>
共生成 N 条测试用例
```

### 示例2：仅提供需求文档

用户输入：
```
需求文档：<需求文档绝对路径>
```

输出：
```
需求文档：<回显用户传入的需求文档路径>
测试用例文件已保存到：<实际生成的xlsx文件路径>
共生成 N 条测试用例
```

### 输出规则
- **需求文档**：回显用户实际传入的需求文档文件路径
- **思维导图**：仅在用户提供了思维导图时回显，未提供则不展示此行
- **测试用例文件已保存到**：回显实际生成并保存的 xlsx 文件完整路径
- **共生成 N 条测试用例**：N 为实际生成的测试用例数量，动态统计

## 注意事项
- 用例编号连续递增，不跳号
- 每条用例必须包含明确的前置条件，便于复现
- 预期结果要具体、可验证，避免模糊描述
- 同一模块的用例集中排列
- 优先级组合场景需覆盖多条件冲突情况
- 异常场景需验证兜底策略和用户提示
