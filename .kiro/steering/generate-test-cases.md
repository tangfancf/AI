# 需求文档/思维导图 生成测试用例 Skill

## 用途
根据需求文档（.md/.txt/.docx/.pdf）和/或思维导图图片生成完整的测试用例 xlsx 文件。

## 输入
用户需要提供以下至少一项：
1. **需求文档路径**（选填）：支持 .md/.markdown/.txt/.docx/.pdf 格式
2. **思维导图图片**（选填）：直接在 Kiro 对话中拖入图片，Kiro 直接看图理解结构

## 输出
- `/Users/mac/Desktop/{文档名称}.xlsx` — 包含完整测试用例的 Excel 文件
- 用例字段：编号、用例标题、前置条件、步骤、预期结果、用例等级

## 核心脚本路径
#[[file:test-cases/generate_test_cases.py]]

## 执行命令
```bash
cd /Users/mac/AI/test-cases
python generate_test_cases.py --doc "/path/to/需求文档.md"
```

或使用位置参数：
```bash
python generate_test_cases.py /path/to/需求文档.md
```

## ⚠️ AI 生成测试用例必须遵守的细粒度规则（核心！）

### 规则 1：逐字段拆分原则
需求文档或思维导图中提到的"列表字段"、"展示字段"、"表单字段"等，**必须逐个字段生成独立的验证用例**。

❌ **错误**（粗略）：
```python
["TC-001", "【通话记录】列表字段完整展示", "...", "检查各字段", "展示字段包括：专员姓名、坐席编号、拨叫号码...", "P1"]
```

✅ **正确**（细粒度）：
```python
["TC-001", "【通话记录-列表字段】专员姓名字段展示外部人员姓名+飞书ID", "...", "查看专员姓名字段", "外部人员显示姓名+飞书ID", "P1"],
["TC-002", "【通话记录-列表字段】坐席编号字段展示", "...", "查看坐席编号字段", "显示拨打时绑定的坐席编号", "P1"],
["TC-003", "【通话记录-列表字段】拨叫号码展示手机号+省份城市+用户ID", "...", "查看拨叫号码字段", "显示手机号+省份城市+用户ID", "P1"],
# ... 每个字段一条
```

### 规则 2：功能操作逐项覆盖原则
- 筛选/搜索功能 → 每个筛选维度单独一条用例
- 按钮/交互操作 → 每个操作单独一条用例
- 不同入口触发同一功能 → 每个入口单独一条用例

❌ **错误**：
```python
["TC-001", "【通话记录】支持多种筛选条件", "...", "使用不同筛选条件", "筛选结果正确", "P1"]
```

✅ **正确**：
```python
["TC-001", "【通话记录-筛选】按专员姓名筛选", "...", "输入专员姓名关键词筛选", "仅展示该专员记录", "P1"],
["TC-002", "【通话记录-筛选】按拨叫号码筛选", "...", "输入手机号筛选", "仅展示该号码记录", "P1"],
["TC-003", "【通话记录-筛选】按通话时间范围筛选", "...", "选择起止时间筛选", "仅展示该时间段记录", "P1"],
["TC-004", "【通话记录-筛选】按接通状态筛选", "...", "选择已接通/未接通筛选", "仅展示对应状态记录", "P1"],
["TC-005", "【通话记录-筛选】多条件组合筛选", "...", "同时设置多个条件", "展示同时满足所有条件的记录", "P2"],
["TC-006", "【通话记录-筛选】重置筛选条件", "...", "点击重置", "所有条件清空，列表恢复全部", "P2"],
```

### 规则 3：历史功能/回归用例原则
需求文档或思维导图中标注为"历史功能"、"回归"、"点位回归"的模块，**必须对其列出的每一个子项（字段、接口、操作、入口）生成验证用例**。

❌ **错误**：
```python
["TC-001", "【SCRM接打回归】各点位拨打功能正常", "...", "各入口拨打", "正常外呼", "P1"]
```

✅ **正确**：
```python
["TC-001", "【SCRM接打回归】销售工作台-外呼拨打正常", "...", "...", "...", "P0"],
["TC-002", "【SCRM接打回归】销售工作台-我的商机拨打正常", "...", "...", "...", "P1"],
["TC-003", "【SCRM接打回归】销售工作台-我的客户拨打正常", "...", "...", "...", "P1"],
["TC-004", "【SCRM接打回归】切换成员拨打正常", "...", "...", "...", "P2"],
["TC-005", "【SCRM接打回归】跟进记录拨打正常", "...", "...", "...", "P2"],
```

### 规则 4：枚举值/多选项逐一验证原则
- 多个厂商 → 每个厂商单独一条用例
- 多个数据源 → 每个数据源单独一条用例
- 多个不入库原因 → 每个原因单独一条用例
- 多种状态 → 每种状态单独一条用例

### 规则 5：最小粒度原则
- 一条用例只验证一个点
- 步骤和预期结果聚焦于单一验证目标
- 如果预期结果超过 5 行，考虑拆分为多条用例

### 规则 6：思维导图模块识别原则
- 思维导图中的每一个"分支"（一级/二级/三级子节点）都代表一个需要覆盖的测试场景
- **不得遗漏任何分支节点**，尤其是叶子节点（最终的具体功能点）
- 每个叶子节点至少对应 1 条测试用例

### 规则 7：总量预期与自检
- 一个中等复杂度需求（含 10+ 功能点），最终用例数通常 **>= 80 条**
- 若生成用例 **< 50 条**，大概率存在遗漏，必须重新审视每个节点
- 生成完成后，逐一对照需求文档/思维导图的所有节点，确认无遗漏

## 用例等级定义

| 等级 | 含义 | 适用场景 |
|------|------|----------|
| P0 | 冒烟/准入 | 核心功能可用性验证，不通过则阻塞测试 |
| P1 | 核心流程 | 主要业务流程、关键字段展示、重要交互 |
| P2 | 分支/边界 | 分支逻辑、边界值、次要字段、筛选功能 |
| P3 | 异常/极端 | 异常数据、接口容错、极端场景 |

## 用例标题格式
- 格式：`【所属模块-子模块】具体验证点描述`
- 示例：`【通话记录-列表字段】专员姓名字段展示外部人员姓名+飞书ID`
- 子模块可选，但多层级模块建议带上以便分类

## 生成流程

### Step 1：读取并分析需求内容
1. 读取需求文档全文（通过脚本或直接读取文件）
2. 查看思维导图图片（用户在对话中直接拖入，Kiro 直接看图理解结构）
3. 提取所有功能模块、字段列表、交互操作、数据源、枚举值

### Step 2：按模块分别生成用例

对每个功能模块，按以下三层结构依次遍历生成用例：

#### 第一层：正向功能覆盖（P0/P1 为主）
按需求文档/思维导图的结构，逐模块生成：

| 维度 | 拆解规则 | 举例 |
|------|----------|------|
| 核心流程 | 主流程 + 各分支路径各一条 | 新增成功、编辑成功、删除成功 |
| 列表/展示字段 | 逐字段独立一条 | 每个列表字段单独验证展示正确性 |
| 筛选/搜索 | 每个筛选维度一条 + 组合筛选 + 重置 | 按姓名筛选、按时间筛选、多条件组合 |
| 枚举/多值场景 | 每个枚举值独立一条 | 每种不入库原因、每个供应商 |
| 配置管理(CRUD) | 增/删/改/查各一条 + 权限控制 | 新增规则、编辑规则、删除规则 |

#### 第二层：分支与边界补充（P2 为主）
在正向覆盖基础上，补充条件分支和临界场景：

| 维度 | 拆解规则 | 举例 |
|------|----------|------|
| 条件分支 | 每个 if/else 判断的每条路径 | 有数据/无数据、已配置/未配置 |
| 优先级/冲突 | 多规则同时命中时的优先级验证 | DP规则 vs FP规则优先级 |
| 边界值 | 阈值上下限、空值、极值 | 0条记录、最大数量、刚好超时 |
| UI交互状态 | 按钮置灰/可用切换、弹窗展示/关闭、loading | 未选择时保存按钮置灰 |
| 兼容性 | 升级后老数据完整性、多入口一致性 | 历史规则升级后仍可正常编辑 |

#### 第三层：异常与容错（P3 为主）
最后补充系统级异常场景：

| 维度 | 拆解规则 | 举例 |
|------|----------|------|
| 接口异常 | 超时、5xx、网络断开各一条 | 不入库接口超时后重试 |
| 并发/竞态 | 同时操作、重复提交 | 多人同时编辑同一规则 |
| 数据异常 | 脏数据、格式错误、超长输入 | 号码位数不足、特殊字符注入 |
| 服务不可用 | 依赖服务挂掉时的降级表现 | 多维表格连接中断 |

#### 第四层：历史功能回归（独立模块）
针对需求中标注"回归"/"历史功能"/"点位回归"的内容：
- 逐入口、逐字段、逐操作分别生成验证用例
- 确保老功能不因新需求上线而受损

> **执行要点**：先完成第一层再进第二层，逐层补充，避免遗漏正向场景就直接跳到异常。每层生成完后回顾需求文档确认无遗漏节点。

### Step 3：合并去重（如有多个来源）
- 若从多个来源分别生成用例，合并后按标题相似度 > 70% 判定重复
- 保留更详细的一条

### Step 4：输出 Excel
- 调用脚本中的 `create_test_cases()` 函数输出到桌面 xlsx 文件
- 格式化输出，表头样式、冻结首行、自动列宽

## 注意事项
- 需要安装 openpyxl：`pip install openpyxl`
- 读取 .docx 需要：`pip install python-docx`
- 读取 .pdf 需要：`pip install pdfplumber`
- 输出路径固定为 `/Users/mac/Desktop/{文档名}.xlsx`
- 同名文件会覆盖，注意备份
- 思维导图图片无需任何 OCR 库，直接在 Kiro 对话中拖入即可
